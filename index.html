Pop Quiz - var, let, const, hoisting

What is the difference between var and let?

var declared variables can get re-declared. Variables declared with var are also accessible in the windows object, window.xxx. var declared variables will hoist to the top of the scope where it is declared. The scope for variables declared with var is the function where it was declared, regardless of code block. Variables declared with var can change.

let declared variables canNOT get re-declared and they also do NOT 'hoist'. Variables declared with let are not added to the window object. The scope for variables declared with let is block scope -- the variable is only known to all the code within the { and } where the variable was declared with let. Variables declared with let can change.

What is the difference between var and const?

Same as above: let declared variables canNOT get re-declared and they also do NOT 'hoist'. Variables declared with let are not added to the window object. The scope for variables declared with let is block scope -- the variable is only known to all the code within the { and } where the variable was declared with let. Variables declared with let can change.

const declared variables function in the same way as let-declared variables except const are CONSTant and cannot change from the value assigned when declared. The big caveat are reference objects -- the properties and values within the object can change but the reference to the object cannot change.

What is the difference between let and const?

Covered above. let-declared variables can change, const-declared things cannot change. (reference-object exception caveat).

What is hoisting?

Hoisting is applicable to var declared variables. With hoisting, the declaration of the var happens at the top of the scope / function regardless of where it is initialized and used. The var is not assigned a value until the assign is encountered in code. The variable is undefined (instead of hard error not declared) when the var-declared variable is referenced in code before it was actually assigned.

Contrasting with let, the declaration occurs where the variable is declared in the code block. Referencing a let-declared variable before it is declared will cause a not declared hard error (not sure if only the function will not run or the entire script file).
